마지막 15차시는 인공지능 통제와 학습 데이터를 학습내용 및 주제로 삼고, 인공지능 통제와 학습 데이터의 중요성에 대한 이해를 학습 목표로 제시했어. 토론은 없고 과제가 있네. 그렇게 K-MOOC강좌가 진행되었나봐.

사실 : ai는 학습 데이터로부터 패턴을 학습함으로 ai의 성능 역시 학습 데이터에 의존적이다. 또한 ai는 인간이 적절한 통제 체계를 마련하여 안전하게 활용해야 한다.

나의 관찰 : 같은 질문이라도 프롬프트를 조금만 수정하거나 프로젝트의 배경 정보를 충분히 제공하면 답변의 품질이 크게 달라졌고, 반대로 불완전하거나 모호한 정보를 제공하면 ai도 기대와 다른 결과를 생성하는 경우가 많았다. 이러한 경험을 통해 ai의 성능은 모델 자체뿐 아니라 입력 정보의 품질에도 크게 영향을 받는다는 점을 체감하였다. 또한 ai를 효과적으로 활용하기 위해서는 모델을 통제하는 능력과 함께 입력 정보를 구조화하고 관리하는 능력도 중요하다고 생각한다. 특히 프로젝트가 커질수록, 1회 사용량이 부족할수록, 시용량을 절감하고 싶을수록 ai의 동선을 치밀하게 설계할 필요가 있다고 느꼈다. 

과제 : 학습 데이터의 중요성과 AI 통제의 필요성을 서술하시오.

gpt가 예상하는 나의 의견 : 생성형 AI는 단순히 성능이 좋은 모델을 사용하는 것보다, 어떤 정보를 제공하고 어떻게 통제하는지가 더 중요하다고 생각한다. 실제 프로젝트에서는 프롬프트, 체크리스트, 인수인계 문서, 원본 보존 등을 통해 AI를 원하는 방향으로 유도하였으며, 이러한 관리 체계가 AI의 활용 결과에 큰 영향을 주었다. 따라서 AI 시대의 핵심 역량은 AI에게 일을 맡기는 것이 아니라, AI가 올바르게 일할 수 있도록 데이터를 준비하고 결과를 검증하며 통제하는 능력이라고 생각한다.

나의 의견 : ai는 학습 데이터로부터 패턴을 학습함으로 ai의 성능 역시 학습 데이터에 의존적이다. 한편 개인적으로 사용해본 결과 같은 질문이라도 프롬프트를 조금만 수정하거나 프로젝트의 배경 정보를 충분히 제공하면 답변의 품질이 크게 달라졌고, 반대로 불완전하거나 모호한 정보를 제공하면 ai도 기대와 다른 결과를 생성하는 경우가 많았다. 또한 프로젝트의 규모가 커질수록 하나의 AI만으로 모든 작업을 수행하기 어려워지고, 무료 사용량이나 작업 연속성 등의 현실적인 제약도 함께 고려해야 했다. 이처럼 프로젝트가 커질수록, 1회 사용량이 부족할수록, 시용량을 절감하고 싶을수록 작업 순서를 설계하고, 필요한 정보를 적절한 시점에 제공하며, 여러 ai의 역할을 조정하는 등 ai의 동선을 치밀하게 설계하고 통제하는 능력이 중요하다고 생각한다.
